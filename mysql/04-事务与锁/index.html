    04 — 事务与锁¶

    4.1 事务基础¶

    什么是事务¶

    事务是一组不可分割的 SQL 操作，要么全部成功，要么全部失败回滚。

    ACID 四大特性¶

    特性 说明
    原子性 Atomicity 事务中的操作要么全做，要么全不做
    一致性 Consistency 事务前后数据保持一致状态
    隔离性 Isolation 并发事务之间互不干扰
    持久性 Durability 事务提交后数据永久保存

    事务语法¶

    -- 开启事务
START TRANSACTION;
-- 或
BEGIN;

-- 执行操作
UPDATE account SET balance = balance - 100 WHERE id = 1;
UPDATE account SET balance = balance + 100 WHERE id = 2;

-- 提交
COMMIT;

-- 回滚
ROLLBACK;

-- 设置保存点
SAVEPOINT sp1;
ROLLBACK TO sp1;  -- 回滚到保存点
RELEASE SAVEPOINT sp1;

    自动提交¶

    -- 查看自动提交状态
SHOW VARIABLES LIKE 'autocommit';
-- MySQL 默认 autocommit = ON（每条语句自动提交）

-- 关闭自动提交（只在当前会话生效）
SET autocommit = 0;
-- 此后所有语句需手动 COMMIT

    4.2 并发问题¶

    问题 描述
    脏读 读到其他事务未提交的修改
    不可重复读 同一事务内两次读取结果不一致（其他事务 UPDATE 了）
    幻读 同一事务内两次查询行数不一致（其他事务 INSERT/DELETE 了）

    4.3 隔离级别¶

    隔离级别 脏读 不可重复读 幻读 默认
    READ UNCOMMITTED 有 有 有 —
    READ COMMITTED 无 有 有 Oracle / PostgreSQL
    REPEATABLE READ 无 无 部分无* MySQL InnoDB
    SERIALIZABLE 无 无 无 —

    MySQL InnoDB 在 REPEATABLE READ 下通过 MVCC + Gap Lock 基本解决了幻读。

    -- 查看隔离级别
SELECT @@transaction_isolation;  -- MySQL 8.0+
SELECT @@tx_isolation;           -- MySQL 5.x

-- 设置隔离级别
SET SESSION TRANSACTION ISOLATION LEVEL READ COMMITTED;
SET GLOBAL TRANSACTION ISOLATION LEVEL REPEATABLE READ;

    4.4 MVCC（多版本并发控制）¶

    核心概念¶

    • 每行数据有多个版本，读写互不阻塞
    • 通过 trx_id（事务 ID）和 roll_pointer（回滚指针）形成版本链
    • 配合 Read View 判断数据的可见性

    Read View 规则¶

    事务 T 读取数据时，判断该行版本是否可见：
1. trx_id < min_trx_id → 已提交，可见
2. trx_id > max_trx_id → 未开始或未提交，不可见
3. min_trx_id ≤ trx_id ≤ max_trx_id → 如果 trx_id 在活跃列表中，不可见

    RC vs RR 的 Read View¶

    • READ COMMITTED：每次 SELECT 都生成新的 Read View
    • REPEATABLE READ：事务开始时生成一次 Read View（快照读）

    4.5 锁机制¶

    锁的分类¶

    按粒度：
  表锁（Table Lock）
  行锁（Row Lock）
  页锁（Page Lock）—— 介于表锁和行锁之间

按模式：
  共享锁（S Lock / 读锁）—— SELECT ... LOCK IN SHARE MODE
  排他锁（X Lock / 写锁）—— INSERT/UPDATE/DELETE 自动加

按算法（InnoDB）：
  记录锁（Record Lock）—— 锁定索引记录
  间隙锁（Gap Lock）—— 锁定索引记录之间的间隙
  临键锁（Next-Key Lock）—— Record + Gap

    加锁示例¶

    -- 共享锁（允许其他事务读，不允许写）
SELECT * FROM student WHERE id = 1 LOCK IN SHARE MODE;
-- 或 (MySQL 8.0+)
SELECT * FROM student WHERE id = 1 FOR SHARE;

-- 排他锁（不允许其他事务读和写）
SELECT * FROM student WHERE id = 1 FOR UPDATE;

-- 排他锁 + 跳过已锁行（8.0+）
SELECT * FROM student WHERE age > 20 FOR UPDATE SKIP LOCKED;

-- 排他锁 + 立即失败（不等锁）
SELECT * FROM student WHERE id = 1 FOR UPDATE NOWAIT;

    死锁¶

    事务 A：锁定行 1 → 等待行 2
事务 B：锁定行 2 → 等待行 1
→ 死锁！

    -- 查看死锁信息
SHOW ENGINE INNODB STATUS;

-- 查看当前锁等待
SELECT * FROM information_schema.INNODB_TRX;
SELECT * FROM information_schema.INNODB_LOCKS;     -- 8.0 前
SELECT * FROM performance_schema.data_locks;          -- 8.0+
SELECT * FROM performance_schema.data_lock_waits;     -- 8.0+

-- 设置锁等待超时
SET innodb_lock_wait_timeout = 50;  -- 默认 50 秒

    避免死锁¶

    • 按相同顺序访问表和行
    • 事务尽量简短
    • 尽量用索引（减少锁范围）
    • 避免在事务中交互等待（用户输入等）
    • 大事务拆小

    4.6 事务最佳实践¶

    -- 优雅的转账事务
START TRANSACTION;

-- 检查余额
SELECT balance FROM account WHERE id = 1 FOR UPDATE;

IF balance >= 100 THEN
    UPDATE account SET balance = balance - 100 WHERE id = 1;
    UPDATE account SET balance = balance + 100 WHERE id = 2;
    COMMIT;
ELSE
    ROLLBACK;
END IF;

-- 乐观锁（版本号）
UPDATE product
SET stock = stock - 1, version = version + 1
WHERE id = 100 AND version = #{current_version};
-- 如果 affected_rows = 0，说明并发冲突，重试
