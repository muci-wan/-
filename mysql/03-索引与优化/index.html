    03 — 索引与优化¶

    3.1 索引基础¶

    什么是索引¶

    索引是帮助 MySQL 高效获取数据的数据结构。类似于书的目录，能快速定位数据而不必全表扫描。

    索引的代价¶

    • 空间：额外占用磁盘空间
    • 时间：增删改操作需要同步维护索引
    • 不是越多越好：索引过多会拖慢写入性能

    3.2 索引类型¶

    B+Tree 索引（默认）¶

    • InnoDB 和 MyISAM 的默认索引
    • 所有数据存储在叶子节点
    • 叶子节点之间通过双向链表连接（范围查询友好）
    • 适合：等值查询、范围查询、排序

    Hash 索引¶

    • Memory 引擎支持
    • 只支持等值查询，不支持范围查询
    • 极快但局限性大

    全文索引（FULLTEXT）¶

    ALTER TABLE article ADD FULLTEXT INDEX ft_content (title, content);
SELECT * FROM article WHERE MATCH(title, content) AGAINST('MySQL 索引');

    空间索引（SPATIAL）¶

    -- 地理数据索引
CREATE SPATIAL INDEX idx_location ON places(location);

    3.3 聚集索引与二级索引¶

    InnoDB 的索引结构¶

    聚簇索引（主键索引）：叶子节点存完整行数据
    ↓
二级索引（普通索引）：叶子节点存主键值
    ↓
回表：通过二级索引找到主键，再去聚簇索引找完整数据

    -- 主键就是聚簇索引
CREATE TABLE t (
    id INT PRIMARY KEY,  -- 聚簇索引
    name VARCHAR(50),
    age INT,
    INDEX idx_name (name) -- 二级索引（存 name → id）
);

-- 查询过程
SELECT * FROM t WHERE name = '张三';
-- 1. 在 idx_name 中找到 name='张三' 对应的 id
-- 2. 用 id 去聚簇索引查完整行（回表）

-- 覆盖索引：避免回表
SELECT id, name FROM t WHERE name = '张三';
-- idx_name 已经包含 id 和 name，不需要回表

    覆盖索引¶

    查询的列都在索引中，不需要回表，性能最好。

    -- 创建复合索引实现覆盖
CREATE INDEX idx_name_age ON student(name, age);

-- 覆盖索引（无需回表）
SELECT name, age FROM student WHERE name = '张三';

    3.4 索引操作¶

    -- 创建索引
CREATE INDEX idx_name ON student(name);
CREATE UNIQUE INDEX idx_email ON student(email);
ALTER TABLE student ADD INDEX idx_age (age);

-- 创建复合索引（最左前缀原则）
CREATE INDEX idx_name_age ON student(name, age);

-- 查看索引
SHOW INDEX FROM student;

-- 删除索引
DROP INDEX idx_name ON student;
ALTER TABLE student DROP INDEX idx_age;

    3.5 最左前缀原则¶

    复合索引 (a, b, c) 的生效规则：

    WHERE 条件 是否使用索引
    a = 1 使用（a）
    a = 1 AND b = 2 使用（a, b）
    a = 1 AND b = 2 AND c = 3 使用（a, b, c）
    a = 1 AND c = 3 使用 a，c 无法使用
    b = 2 不使用
    b = 2 AND c = 3 不使用

    复合索引要按查询频率和过滤性来安排字段顺序：高区分度字段放前面。

    3.6 Explain 执行计划¶

    EXPLAIN SELECT * FROM student WHERE name = '张三';

    关键字段解读¶

    字段 含义
    id 查询序号（越大越先执行）
    select_type SIMPLE / PRIMARY / SUBQUERY / DERIVED 等
    type 访问类型（性能从好到差）
    possible_keys 可能使用的索引
    key 实际使用的索引
    key_len 使用索引的长度
    rows 预估扫描行数
    Extra 额外信息

    type 性能排序（优→劣）¶

    system > const > eq_ref > ref > range > index > ALL

system  : 系统表，仅一行
const   : 主键或唯一索引等值查询
ref     : 普通索引等值查询
range   : 索引范围查询（>, <, BETWEEN, IN）
index   : 全索引扫描
ALL     : 全表扫描（最差，必须优化）

    Extra 重要信息¶

    Using index         → 覆盖索引，很好
Using where         → 使用了 WHERE 过滤
Using index condition → 索引条件下推（ICP）
Using temporary     → 使用了临时表（需优化）
Using filesort      → 文件排序（需优化）

    3.7 索引优化策略¶

    哪些列适合建索引¶

    • WHERE 条件经常出现的列
    • JOIN 关联的列
    • ORDER BY / GROUP BY 的列
    • 区分度高的列（如用户 ID，而非性别）

    哪些列不适合建索引¶

    • 很少参与查询的列
    • 区分度极低的列（如 gender 只有 M/F）
    • 频繁更新的列（维护开销大）
    • 大字段（TEXT、BLOB）

    索引失效场景¶

    -- 1. 对索引列使用函数或运算
SELECT * FROM student WHERE YEAR(birthday) = 2000;  -- 失效
SELECT * FROM student WHERE birthday >= '2000-01-01'; -- 有效

-- 2. 隐式类型转换
SELECT * FROM student WHERE phone = 13800138000;  -- phone 是 VARCHAR，失效
SELECT * FROM student WHERE phone = '13800138000'; -- 有效

-- 3. LIKE 以 % 开头
SELECT * FROM student WHERE name LIKE '%三';       -- 失效
SELECT * FROM student WHERE name LIKE '张%';       -- 有效

-- 4. OR 条件中有非索引列
SELECT * FROM student WHERE id = 1 OR name = '王五'; -- name 无索引则全表扫描

-- 5. !=   <>   NOT IN   NOT EXISTS
SELECT * FROM student WHERE age != 20;              -- 可能不走索引

-- 6. IS NULL 或 IS NOT NULL（视情况）
SELECT * FROM student WHERE email IS NOT NULL;     -- 可能不走索引

    3.8 SQL 优化要点¶

    查询优化¶

    -- 用具体列代替 SELECT *
SELECT id, name FROM student WHERE age > 20;  -- 好
SELECT * FROM student WHERE age > 20;          -- 差

-- 分页优化（大偏移量）
SELECT * FROM student LIMIT 100000, 20;       -- 差：扫描 100020 行
SELECT * FROM student WHERE id > 100000 LIMIT 20; -- 好：定位后取 20 行
-- 或使用延迟关联
SELECT * FROM student s
JOIN (SELECT id FROM student LIMIT 100000, 20) t ON s.id = t.id;

-- IN vs EXISTS
-- 子查询结果集小用 IN，外表小用 EXISTS
SELECT * FROM student WHERE id IN (SELECT student_id FROM score WHERE score > 90);
SELECT * FROM student s WHERE EXISTS (SELECT 1 FROM score sc WHERE sc.student_id = s.id);

-- 避免在 WHERE 中对字段做计算
SELECT * FROM student WHERE age * 2 > 40;     -- 差
SELECT * FROM student WHERE age > 20;          -- 好

    表设计优化¶

    • 字段尽量 NOT NULL，避免 NULL 判断
    • 用合适的最小数据类型（如年龄用 TINYINT）
    • 用整型做主键（InnoDB 聚簇索引）
    • 避免在表中存储大文本，拆到单独表
