# 06 — 运维管理

## 6.1 用户与权限

### 用户管理

```sql
-- 创建用户
CREATE USER 'dev'@'localhost' IDENTIFIED BY 'StrongP@ss123';
CREATE USER 'dev'@'%' IDENTIFIED BY 'StrongP@ss123';  -- 任意主机

-- 修改密码
ALTER USER 'dev'@'localhost' IDENTIFIED BY 'NewP@ss456';
-- 或 (MySQL 8.0+)
SET PASSWORD FOR 'dev'@'localhost' = 'NewP@ss456';

-- 删除用户
DROP USER 'dev'@'localhost';

-- 查看用户
SELECT user, host FROM mysql.user;

-- 锁定/解锁用户
ALTER USER 'dev'@'localhost' ACCOUNT LOCK;
ALTER USER 'dev'@'localhost' ACCOUNT UNLOCK;

-- 密码过期策略
ALTER USER 'dev'@'localhost' PASSWORD EXPIRE INTERVAL 90 DAY;
ALTER USER 'dev'@'localhost' PASSWORD EXPIRE NEVER;
```

### 权限管理

```sql
-- 授予权限
GRANT SELECT, INSERT, UPDATE ON mydb.* TO 'dev'@'localhost';
GRANT ALL PRIVILEGES ON mydb.* TO 'admin'@'localhost';
GRANT SELECT ON mydb.student TO 'reader'@'%';

-- 撤销权限
REVOKE INSERT ON mydb.* FROM 'dev'@'localhost';
REVOKE ALL PRIVILEGES ON mydb.* FROM 'dev'@'localhost';

-- 查看权限
SHOW GRANTS FOR 'dev'@'localhost';
SHOW GRANTS FOR CURRENT_USER();

-- 刷新权限（修改权限后执行）
FLUSH PRIVILEGES;
```

### 常用权限表

| 权限 | 说明 |
|------|------|
| ALL PRIVILEGES | 所有权限 |
| SELECT | 查询 |
| INSERT | 插入 |
| UPDATE | 更新 |
| DELETE | 删除 |
| CREATE | 创建表 |
| DROP | 删除表 |
| ALTER | 修改表 |
| INDEX | 创建索引 |
| REFERENCES | 创建外键 |
| CREATE VIEW | 创建视图 |
| EXECUTE | 执行存储过程 |
| FILE | 读写文件（高危） |
| SUPER | 超级权限（高危） |

### 角色（8.0+）

```sql
-- 创建角色
CREATE ROLE 'app_dev', 'app_read';

-- 给角色授权
GRANT SELECT, INSERT, UPDATE ON mydb.* TO 'app_dev';
GRANT SELECT ON mydb.* TO 'app_read';

-- 把角色赋给用户
GRANT 'app_dev' TO 'dev1'@'%';

-- 设置默认角色
SET DEFAULT ROLE 'app_dev' TO 'dev1'@'%';

-- 激活角色（客户端）
SET ROLE 'app_dev';
```

## 6.2 备份与恢复

### mysqldump 逻辑备份

```bash
# 备份单个数据库
mysqldump -u root -p mydb > mydb_backup.sql

# 备份多个数据库
mysqldump -u root -p --databases db1 db2 > dbs_backup.sql

# 备份所有数据库
mysqldump -u root -p --all-databases > all_backup.sql

# 仅备份结构（不含数据）
mysqldump -u root -p --no-data mydb > mydb_schema.sql

# 仅备份数据（不含结构）
mysqldump -u root -p --no-create-info mydb > mydb_data.sql

# 备份特定表
mysqldump -u root -p mydb student score > tables_backup.sql

# 带存储过程和事件
mysqldump -u root -p --routines --events mydb > mydb_full.sql
```

### 恢复

```sql
-- 在 MySQL 中
SOURCE /path/to/backup.sql;

-- 或命令行
mysql -u root -p mydb < mydb_backup.sql
```

### 二进制日志备份（PITR）

```bash
# 查看 binlog 状态
mysql -u root -p -e "SHOW MASTER STATUS"

# 恢复到指定时间点
mysqlbinlog --start-datetime="2026-06-20 10:00:00" \
            --stop-datetime="2026-06-20 10:30:00" \
            /var/lib/mysql/binlog.000001 | mysql -u root -p
```

### 物理备份

```bash
# XtraBackup 热备份（推荐用于大库）
xtrabackup --backup --target-dir=/backup/full
xtrabackup --prepare --target-dir=/backup/full
xtrabackup --copy-back --target-dir=/backup/full
```

## 6.3 日志系统

```sql
-- 错误日志
SHOW VARIABLES LIKE 'log_error';

-- 慢查询日志
SET GLOBAL slow_query_log = ON;
SET GLOBAL long_query_time = 2;  -- 超过 2 秒记录

-- 查看慢查询日志位置
SHOW VARIABLES LIKE 'slow_query_log_file';

-- binlog
SHOW BINARY LOGS;
SHOW MASTER STATUS;

-- 通用查询日志（生产慎开）
SET GLOBAL general_log = ON;
```

## 6.4 性能监控

```sql
-- 查看当前连接
SHOW PROCESSLIST;
-- 或
SELECT * FROM information_schema.PROCESSLIST;

-- 杀掉连接
KILL 123;  -- 连接 ID

-- 查看状态变量
SHOW STATUS LIKE 'Threads%';
SHOW STATUS LIKE 'Connections';
SHOW STATUS LIKE 'Questions';
SHOW STATUS LIKE 'Slow_queries';

-- InnoDB 引擎状态
SHOW ENGINE INNODB STATUS\G

-- 表空间使用
SELECT
    table_schema AS '数据库',
    table_name AS '表名',
    ROUND(SUM(data_length + index_length) / 1024 / 1024, 2) AS '大小(MB)'
FROM information_schema.tables
WHERE table_schema NOT IN ('mysql','information_schema','performance_schema','sys')
GROUP BY table_schema, table_name
ORDER BY SUM(data_length + index_length) DESC
LIMIT 20;
```

## 6.5 主从复制

### 基本概念

```
Master（主库）: 写操作 → 生成 binlog
    ↓ (IO 线程读取)
Relay Log（中继日志）
    ↓ (SQL 线程回放)
Slave（从库）: 只读
```

### 配置主库

```ini
# my.cnf [mysqld]
server-id = 1
log-bin = mysql-bin
binlog_format = ROW
```

### 配置从库

```ini
# my.cnf [mysqld]
server-id = 2
relay-log = mysql-relay-bin
read_only = 1
```

### 搭建复制

```sql
-- 主库创建复制账号
CREATE USER 'repl'@'%' IDENTIFIED BY 'repl_pass';
GRANT REPLICATION SLAVE ON *.* TO 'repl'@'%';

-- 获取主库 binlog 位置
SHOW MASTER STATUS;
-- 记下 File 和 Position

-- 从库配置
CHANGE MASTER TO
    MASTER_HOST = '主库IP',
    MASTER_USER = 'repl',
    MASTER_PASSWORD = 'repl_pass',
    MASTER_LOG_FILE = 'mysql-bin.000001',
    MASTER_LOG_POS = 123;

-- 启动复制
START SLAVE;

-- 查看状态
SHOW SLAVE STATUS\G
-- 关注：Slave_IO_Running 和 Slave_SQL_Running 都应为 Yes
```

### 复制延迟排查

```sql
-- 从库查看延迟秒数
SHOW SLAVE STATUS\G;  -- Seconds_Behind_Master

-- 常见原因
-- 1. 从库硬件差
-- 2. 大量写入导致 binlog 积压
-- 3. 大事务阻塞 SQL 线程
-- 4. 从库有其他负载
```

## 6.6 常用命令速查

```bash
# 连接
mysql -u root -p -h localhost -P 3306

# 导出
mysqldump -u root -p mydb > mydb.sql

# 导入
mysql -u root -p mydb < mydb.sql

# 查看版本
mysql --version
mysql -u root -p -e "SELECT VERSION()"

# 查看配置
mysql -u root -p -e "SHOW VARIABLES LIKE '%buffer_pool%'"

# 不进入交互模式执行 SQL
mysql -u root -p -e "SELECT COUNT(*) FROM mydb.student"
```
